## Vetted findings that did NOT get plans (backlog, in rough priority order)

- **innerHTML i18n rendering** (`src/app/features/instructions/instructions.ts:14,15,20,41`):
  four `[innerHTML]` bindings render i18n strings containing `<strong>`.
  Angular sanitizes the binding and the strings are developer-controlled, so
  severity is low; defense-in-depth fix is to move the markup into the
  template. S effort.
- **ESLint missing**: no lint target at all (CLAUDE.md confirms). `ng add
  @angular-eslint/schematics` + baseline run. M effort, additive.
- **`as unknown as` casts on Supabase joins** (match.service.ts:94,106,
  bet.service.ts:103, leaderboard.service.ts:45, match-card.ts:151): the
  generated types likely can express the join shapes (supabase-js infers
  embedded-resource types); removing the casts restores compiler coverage of
  schema drift. M effort, MED confidence — investigate first.
- **SSR pages uncacheable**: every request hits Supabase for the full match
  list; anonymous SSR output is identical for all users, so
  `Cache-Control: s-maxage=60, stale-while-revalidate` on document responses
  would cut DB load on traffic spikes. M effort, MED risk (staleness during
  live matches; `Vary: Accept-Language` needed because of Transloco SSR
  detection). Design before doing.
- **Unused `requireUsername` guard** (`src/app/core/guards/require-username.guard.ts`):
  wired to zero routes; the bet dialog duplicates its logic in `open()`.
  Applying it to browsing routes would be *wrong* (browsing is deliberately
  anonymous), so either delete it or leave it for a future gated route. S.
- **Supabase auth advisor**: "leaked password protection" disabled — the app
  uses magic-link/OAuth only (no passwords), so impact ~0; it's a one-toggle
  dashboard change if password auth is ever enabled.

## Findings considered and rejected

(So nobody re-audits them.)

- **"Redundant 60 s polling alongside realtime" (bet-now.ts:123–127)**: the
  code comments mark it belt-and-suspenders for realtime gaps (reconnects,
  missed events) and it only fires while a match is live. By design.
- **"Both EN+ES dictionaries shipped to every client"**: ~2–3 KB gzipped
  total; lazy-splitting risks flash-of-untranslated-text. Not worth it.
- **".mcp.json exposes project_ref"**: the Supabase project ref is public by
  design (it's in every client URL).
- **"Host header promotion in api/index.mjs lacks validation"**: Vercel sets
  `x-forwarded-host` itself, and Angular validates the promoted host against
  `angular.json` `security.allowedHosts` afterwards. Layers exist; the
  pattern is the documented workaround for Angular 21 SSR deopt.
- **"Supabase CORS open to all origins"**: Supabase's model — the anon key is
  public and RLS is the enforcement layer. Architecture, not a bug.
- **"`getSession().then()` without catch" (auth.service.ts:25)**: supabase-js
  `getSession()` resolves `{data, error}` rather than rejecting; nothing to
  catch in practice.
- **Various stylistic flags** (Map non-null assertion in
  `match.service.ts:62`, `getRawValue()` typing in bet-dialog, fire-and-forget
  `navigateByUrl` followed by `return`): code is correct as written; churn
  without payoff.
- **Angular 22 migration**: real (v21 is one major behind) but a framework
  major bump mid-tournament on an app with (currently) two tests is the wrong
  week. Revisit after the final, after plans 001/002 land.
- **Google OAuth completion**: blocked on dashboard/Google Cloud console
  steps only (`signInWithGoogle()` and the callback route already exist);
  it's an operator config task, not a code plan. Instructions were provided
  in the prior session.
